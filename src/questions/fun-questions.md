## What is a cool project that you've recently worked on?

I built a real-time collaborative code editor. It lets multiple people type in the same file simultaneously, which means watching someone else introduce a bug in real time and not being able to physically stop them. It uses WebSockets and operational transformation to merge edits without everything exploding. The real challenge was not the code, it was resisting the urge to add a feature that plays a sad trombone sound every time someone pushes broken code. My coworkers vetoed that one.

## What are some things you like about the developer tools you use?

I like that VS Code has approximately 47,000 extensions and I only actually use four of them (it's a lie, I use 100 at least), but I feel very prepared knowing the other 46,996 exist (I don't). The integrated terminal means I can have five terminal tabs open and forget what I was doing in all of them simultaneously.

Browser DevTools are incredible. I can simulate a slow 3G connection, throttle CPU performance, and then act surprised when my site loads slowly under conditions I deliberately made terrible. The real joy is spending an hour debugging a layout issue only to realize I forgot to close a div. Every time.

I also appreciate that modern testing tools run so fast that I can find out I broke something in under 200 milliseconds instead of waiting minutes like the old days. Efficiency.

## Who inspires you in the front-end community?

I am inspired by the person who writes incredibly detailed blog posts about some obscure CSS property I have never heard of, making me briefly believe I will absolutely use `shape-outside` in my next project. I won't. But I could.

The real heroes are open-source maintainers who merge pull requests at 2 AM and respond to issues with remarkable patience, even when the issue is just "it doesn't work" with no reproduction steps. Their self-control is something I aspire to.

I also admire developers who confidently claim they know how `z-index` works. That level of self-deception is aspirational. F*** frontend. I'm going to conquer the machine learning field.

## Do you have any pet projects? What kind?

I have a personal website that I have redesigned approximately seven times without ever actually publishing content on it. The design is immaculate. There are no words.

I also built a CLI tool that automates project setup. It saves me about 30 seconds per project, which means after a few thousand more projects, it will have officially paid back the time I spent building it. I am very close. Probably.

I have a half-finished component library that I started to learn more about accessibility patterns. It has beautifully documented components and no consumers. It is a monument to good intentions and questionable time management.
